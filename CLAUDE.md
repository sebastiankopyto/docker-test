# Zasady pracy w tym projekcie

## Git — WAŻNE
- **Nie commituj** żadnych zmian. Commity robi wyłącznie użytkownik samodzielnie.
- **Nie manipuluj historią gita** (żadnych `commit`, `push`, `rebase`, `reset`,
  `amend`, `merge`, zmian gałęzi itp.).
- Możesz swobodnie tworzyć i edytować pliki — samo wersjonowanie zostaw użytkownikowi.
